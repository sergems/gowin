Bonus Ball Modes

For every ticket the player must choose one of three bonus options.

Option 1

Include Bonus Ball

Meaning:
Selections are matched including the bonus ball.

Example:

Player picks

12
18
25
31

Bonus Included

If the player matches the required numbers including the bonus ball, payout is based on the "Included Bonus" table.

Option 2

Exclude Bonus Ball

The bonus ball is ignored.

Example

Player chooses

7
14
28

No bonus considered.

Payout comes from the Excluded Bonus table.

Option 3

Bonus Ball Only

Player only bets on the Bonus Ball.

They select ONE number only.

Example

Bonus Ball

17

Only the Bonus Ball determines winnings.

Technical Requirements
Maintain compatibility with the existing PHP codebase and database.
Follow the current MVC/project architecture.
Avoid hardcoded payout values; load all odds from the database.
Validate all inputs on both the client and server.
Ensure tickets remain compatible with existing draw processing.
Design the update so that additional lottery games (e.g., Powerball, Daily Lotto, Mega Lotto) can each have their own independent payout configuration using the same system.
