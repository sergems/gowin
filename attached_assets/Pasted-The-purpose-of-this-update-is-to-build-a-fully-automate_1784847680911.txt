The purpose of this update is to build a fully automated Lottery Scraper Engine that periodically checks official lottery websites, imports new results into the database, and automatically settles all bets.

OBJECTIVE

Build a modular scraping engine that:

Scrapes official lottery websites
Detects newly published results
Saves them into the database
Prevents duplicate imports
Automatically settles all pending tickets
Credits winners
Logs every operation
Can easily support additional lotteries later

No API integration is required at this stage.

PERFORMANCE

Only scrape lotteries scheduled to draw that day.

Skip inactive lotteries.

Use HTTP timeouts.

Retry failed requests once.

Cache downloaded pages during execution.

Prevent concurrent scraper execution.

SECURITY

Sanitize all scraped values.

Validate numbers.

Validate dates.

Use database transactions.

Prevent duplicate settlement.

CODE QUALITY
Follow PSR-12 coding standards.
Use a service-oriented architecture.
Keep scraping logic separate from settlement logic.
Do not place business logic inside controllers.
Use dependency injection where possible.
Return consistent JSON responses from backend endpoints.
Add detailed comments for maintainability.
Make the system easy to extend for additional lotteries and countries.

Important: Build the scraper engine as a generic, reusable framework rather than a one-off scraper. The scraper should be configurable via the database so that future lotteries can be added without modifying the core engine. Every successful import must automatically trigger ticket settlement, ensuring the lottery system remains fully automated.
